# 贡献指南

感谢您有兴趣为 FreeRoom 安全聊天室项目做出贡献！本文档提供了有关如何参与该项目的指南。

## 🚀 快速开始

1. Fork 仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m ':sparkles: Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request

## 🎯 代码风格

### JavaScript
- 使用 4 个空格缩进
- 函数和变量使用 camelCase 命名
- 遵循 ESLint 规范
- 注释清晰说明复杂逻辑

### PHP
- 使用 PSR-4 自动加载标准
- 遵循 PSR-12 代码风格
- 适当添加注释说明复杂逻辑

### HTML/CSS
- 使用语义化标签
- CSS 类名使用 kebab-case
- 遵循 BEM 方法论（如适用）

## 🧪 测试

在提交更改前，请确保：

- 所有功能按预期工作
- 代码没有明显的安全漏洞
- 用户界面在不同设备上正常显示

## 📝 提交信息规范

我们使用 Emoji Log 来规范提交信息：

- `:sparkles:` `:tada:` - 新功能
- `:bug:` - 修复 bug
- `:memo:` - 文档更新
- `:art:` - 代码格式化
- `:recycle:` - 重构代码
- `:zap:` - 性能改进
- `:construction:` - 工作进行中
- `:green_heart:` - 修复 CI 构建
- `:lock:` - 安全相关
- `:arrow_up:` - 升级依赖

## 🚩 提交前检查清单

- [ ] 代码遵循项目编码标准
- [ ] 所有新功能都有适当测试
- [ ] 所有测试均已通过
- [ ] 更改不会破坏现有功能
- [ ] 代码已充分注释

## 🤝 拉取请求流程

1. 确保 PR 描述清楚说明更改内容
2. 提供屏幕截图（如适用）
3. 确保 PR 只解决一个问题/添加一个功能
4. 遵循模板（如适用）

## 📞 联系方式

如有问题，请通过 GitHub Issues 联系项目维护者。
